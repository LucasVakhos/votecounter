# Stage 43.3 - устойчивость к перестройке базы

Код адаптирован под ручную перестройку `VoteCounter.db`.

Что сделано:
- `OpenConnection()` теперь гарантирует наличие папки `Database`;
- при каждом новом SQLite-соединении повторно выполняется безопасная проверка схемы;
- `CREATE TABLE IF NOT EXISTS` и `EnsureColumn` больше не зависят только от старого флага `_initialized`;
- если база была удалена, заменена или пересобрана во время работы приложения, следующий доступ к базе дотянет таблицы и колонки актуальной схемы;
- версия проекта поднята до `1.43.3-db-rebuild-safe`.

Проверка:
- `dotnet build --no-restore -p:RestorePackagesPath="C:\Users\User\.nuget\packages" -p:NuGetAudit=false`
- результат: 0 ошибок, 0 предупреждений.
