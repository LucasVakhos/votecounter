# STAGE20 - DevExpress Layout Pro

Чекпойнт переводит основные таблицы VoteCounter с обычного `DataGridView` на DevExpress `GridControl + GridView`.

## Что изменилось

- Список конкурсов теперь DevExpress GridView.
- Быстрый просмотр "Работы / рейтинг" теперь DevExpress GridView.
- Быстрый просмотр "Голосующие / контроль" теперь DevExpress GridView.
- Таблица голосов на вкладке "Голосование" теперь DevExpress GridView.
- Таблица "Работы конкурса" теперь DevExpress GridView с редактированием и добавлением строк.
- Таблица "Голосующие / судьи" теперь DevExpress GridView с редактированием и добавлением строк.
- Таблица контроля голосования теперь DevExpress GridView.
- Таблица итогов теперь DevExpress GridView.

## Удобства DevExpress

- В каждой таблице включён поиск.
- Включены фильтры, сортировка и меню колонок.
- Разрешено перетаскивание колонок.
- Редактируемые таблицы работ и судей оставлены редактируемыми.
- Подсветка должников, неизвестных голосующих и победителей перенесена на DevExpress `RowStyle`.

## Layout INI

В STAGE20.1 layout больше не хранится рабочим файлом рядом с exe. Полный ini-текст хранится в базе:

```text
VoteCounter/Database/VoteCounter.db -> Setting.Ini (BLOB)
```

В этом ini сохраняются форма, вкладки, split-панели и layout DevExpress-гридов:

- ширина колонок;
- порядок колонок;
- видимость колонок;
- закрепление колонок;
- сортировка;
- группировка;
- активный фильтр.

## Что не ломалось

- SQLite база остаётся в папке проекта: `VoteCounter/Database/VoteCounter.db`.
- Импорт старых `*.fdb` сохранён.
- Spellchecker при приёме работ сохранён.
- Отчёты и Excel-экспорт сохранены.
