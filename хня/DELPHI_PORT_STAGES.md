# Этапы переноса Rhyme Machine / Delphi в VoteCounter

## STAGE1 - Delphi Core Parser
Разбор строк голосования, разные тире, комментарии, повтор оценки за один номер.

## STAGE2 - Vote Control
Контроль голосующих: кто проголосовал, кто должник, кто неизвестный, пропущенные номера.

## STAGE3 - Rhyme Machine DB Mirror
Локальное JSON-зеркало структуры Firebird/Rhyme Machine.

## STAGE4 - No NuGet Local DB
Откат от SQLite из-за отсутствия пакета `Microsoft.Data.Sqlite` без интернета.

## STAGE5 - Vote Rules
Правила голосования: самоголос = 0, базовая/максимальная оценка, лимиты максимальных.

## STAGE6 - Results
Рейтинг, места, победители, итоговый протокол.

## STAGE7 - Excel PRO
Листы `Рейтинг`, `Протокол`, `Контроль` в итоговом Excel.

## STAGE8 - Work Import
Импорт работ и авторов из текста конкурса или шаблона голосования.


## STAGE9 - режимы авторства

Добавлено разделение сценариев:

- **Учет ведет ведущий** - авторы известны, импорт работ сразу берёт автора из строки: `01 - Название - Автор`.
- **Сторонний счетчик до раскрытия** - авторы скрыты, импорт ставит `Неизвестный автор`: `01 - Название - Неизвестный автор`.
- **После раскрытия** - отдельный режим импорта применяет авторство к существующим работам по номеру: `01 - Автор` или `01 - Название - Автор`.

`Неизвестный автор` больше не затирает уже раскрытого автора при повторном импорте.


## STAGE10 - приём одной работы ведущим

Перенесён рабочий сценарий ведущего: автор присылает полное стихотворение отдельно, ведущий назначает автора и номер.
Первая строка текста превращается в название работы, последняя скобка в заголовке - в тему.
Полный текст сохраняется в ContestWork.Content и в RM-зеркале как Rhyme.RContent.


## STAGE11 - навигатор конкурсов и уточнение приёма работ

- первая вкладка теперь **"Конкурсы"**;
- список показывает №, название, количество работ, судей, принятых голосов, режим авторства и дату обновления;
- один клик выбирает конкурс, двойной клик открывает рабочую вкладку;
- добавлены кнопки "Открыть выбранный", "Новый конкурс", "Удалить конкурс", "Обновить список";
- приём одной работы сверён с Delphi-примером: автор назначается отдельно, тема отдельно, полный текст хранится в содержимом работы.


## STAGE14 - DATABASE STORAGE

По новому требованию хранение переведено в SQLite. Файл базы создаётся в папке проекта: `VoteCounter/Database/VoteCounter.db`. Старые JSON-файлы используются только для первичной миграции.
