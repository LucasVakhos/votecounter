# STAGE38 - Нормальная модель стадий конкурса

Что добавлено:

- у конкурса появились поля жизненного цикла:
  - `StartedAt`;
  - `ClosedAt`;
  - `IsActive`;
- SQLite-схема поднята до версии `21`;
- старые базы автоматически получают новые колонки при старте;
- при загрузке/сохранении поддерживается один активный конкурс;
- завершённый конкурс больше не считается активным;
- новый конкурс становится активным, а прежние уходят в архивный статус;
- при закрытии конкурса заполняется `ClosedAt`, создаётся следующий конкурс со стадией `1 - приём тем`;
- список конкурсов показывает статус `активный` / `архив`;
- активный конкурс поднимается в списке первым;
- карточка конкурса показывает статус, дату начала и дату закрытия.

Проверка:

- `dotnet build --no-restore -p:RestorePackagesPath="C:\Users\User\.nuget\packages" -p:NuGetAudit=false`
- результат: 0 ошибок, 0 предупреждений.
