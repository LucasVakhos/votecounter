# STAGE20.4 - Database Persistence Fix

Исправление подозрения, что база пересоздаётся при каждом новом чекпойнте.

## Что найдено

Код `LocalDatabase.Initialize()` не удалял базу: он только создавал папку `Database` и выполнял `CREATE TABLE IF NOT EXISTS`.

Но ZIP-чекпойнт содержал файл:

```text
VoteCounter/Database/VoteCounter.db
```

Если распаковать новый ZIP поверх существующей рабочей папки с заменой файлов, этот файл мог затереть реальную рабочую базу.

## Что изменено

- `VoteCounter.db` удалён из ZIP-чекпойнта.
- В `.csproj` добавлены явные исключения для `Database/*.db`, `*.db-wal`, `*.db-shm`, `Backups`, `Reports`.
- Автосоздание демонстрационного конкурса `001 - Конкурс картин` отключено.
- Пустая база теперь остаётся пустой, пока пользователь сам не создаст или не импортирует конкурс.
- База остаётся на прежнем месте:

```text
VoteCounter/Database/VoteCounter.db
```

## Как обновляться безопасно

Перед распаковкой нового ZIP лучше не удалять папку `VoteCounter/Database`.
Но теперь даже при обычной распаковке поверх проекта новый ZIP не содержит `VoteCounter.db`, поэтому рабочая база не должна затираться архивом.
