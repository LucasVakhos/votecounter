# STAGE15 - FIREBIRD IMPORT + SPELLCHECK

## Главное изменение

JSON больше не используется как путь миграции старых конкурсов. Рабочее хранение остаётся в SQLite-файле проекта:

```text
VoteCounter/Database/VoteCounter.db
```

Старые конкурсы импортируются из Firebird `*.fdb`.

## Импорт Firebird

Кнопка на вкладке **Конкурсы**:

```text
Импорт старой *.fdb
```

Поддерживаемая старая схема RhymeMachine/Delphi:

- `CONCURSES`
- `CONCURSES_NAMES`
- `CONCURS_DTL`
- `POETS`
- `RHYMES`
- `CONCURSES_NAMES_MEMBERS`
- `CONCURS_VOTES`
- `VOTES`
- `CONCURS_TOPICS`, если есть

Импортируются:

- конкурсы;
- правила голосования;
- работы;
- авторы;
- голосующие;
- старые голоса.

Повторный импорт той же базы обновляет ранее импортированные конкурсы, потому что ID строится детерминированно от пути `*.fdb` и старого `C_ID`.

## Подключение

Добавлен пакет:

```xml
<PackageReference Include="FirebirdSql.Data.FirebirdClient" Version="10.3.4" />
```

Импорт пробует варианты подключения:

- Firebird Server / UTF8;
- Firebird Server / WIN1251;
- Firebird Embedded / UTF8;
- Firebird Embedded / WIN1251.

Если сервер не установлен, для embedded-режима может понадобиться нативный `fbclient.dll`.

## Проверка работ

Добавлен сервис:

```text
Services/WorkSpellChecker.cs
```

Он вызывается перед сохранением в двух местах:

- **Принять работу**;
- **Импорт работ из текста**.

Проверка лёгкая и локальная, без сетевых запросов. Из загруженного IntelliJ-плагина взята только идея отдельного шага проверки перед сохранением, а не весь плагин.

Проверяется:

- наличие названия и текста;
- смешанная русско-латинская раскладка;
- незакрытые кавычки/скобки;
- длинные тире и кавычки-ёлочки;
- табы и хвостовые пробелы;
- повтор знаков `!!`, `??`, `,,`, `;;`;
- очень длинные строки;
- небольшой список частых опечаток.
