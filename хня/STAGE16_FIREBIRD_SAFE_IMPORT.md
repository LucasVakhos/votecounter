# STAGE16 - Firebird Safe Import

Цель этапа: сделать импорт старых `*.fdb` безопасным для рабочей базы `Database/VoteCounter.db`.

## Что добавлено

- Кнопка **"Импорт старой *.fdb"** теперь открывает мастер импорта.
- Перед записью в рабочую базу доступен **предпросмотр**:
  - режим подключения к Firebird;
  - количество найденных конкурсов;
  - количество работ;
  - количество голосующих;
  - количество голосов;
  - предупреждения по отсутствующим таблицам.
- Можно выбрать галочками, какие конкурсы импортировать.
- При повторном импорте защита от дублей работает по:
  - стабильному импортному ID;
  - номеру и названию конкурса, если включена опция.
- Перед применением импорта создаётся резервная копия рабочей SQLite-базы:

```text
VoteCounter/Database/Backups/VoteCounter_yyyyMMdd_HHmmss_before_fdb_import.db
```

- Старые голоса можно:
  - полностью заменить импортированными;
  - или аккуратно слить с уже существующими, обновляя только совпавшие пары "голосующий + работа".
- При импорте можно применить правило конкурса:

```text
голос автора за свою работу = 0
```

## Важно

Рабочая база остаётся в папке проекта:

```text
VoteCounter/Database/VoteCounter.db
```

JSON-хранение не используется.
