# Rhymers STAGE8

WinForms-приложение для приёма голосований и формирования итогового Excel.

## Новое в STAGE8

Добавлен импорт работ/авторов из текста:

- `01 - Название - Автор`
- `01. Название / Автор`
- `01` + следующая строка название + следующая строка автор
- шаблон голосования по темам, где номера работ извлекаются из строк `01 - 3`

Кнопка находится на вкладке **Настройки конкурса**.



## STAGE9 - режимы авторства

Добавлено разделение сценариев:

- **Учет ведет ведущий** - авторы известны, импорт работ сразу берёт автора из строки: `01 - Название - Автор`.
- **Сторонний счетчик до раскрытия** - авторы скрыты, импорт ставит `Неизвестный автор`: `01 - Название - Неизвестный автор`.
- **После раскрытия** - отдельный режим импорта применяет авторство к существующим работам по номеру: `01 - Автор` или `01 - Название - Автор`.

`Неизвестный автор` больше не затирает уже раскрытого автора при повторном импорте.


## STAGE10 - приём одной работы ведущим

Добавлен сценарий для ведущего конкурса:

- поэты присылают работы ведущему по одной;
- ведущий нажимает **Принять работу** на вкладке **Настройки конкурса**;
- вставляет полный текст стихотворения;
- вручную указывает автора;
- программа сама берёт название из первой содержательной строки;
- если заголовок выглядит как `ПОЭТ ВО СНЕ И НАЯВУ (Тема)`, тема берётся из скобок;
- номер работы назначается автоматически, но его можно изменить;
- полный текст работы сохраняется в локальных данных и RM-зеркале.

Этот режим нужен именно тогда, когда учёт ведёт ведущий и авторы ему известны до закрытия конкурса. Для стороннего счётчика остаётся режим `Неизвестный автор` и последующее раскрытие авторства.


## STAGE11 - навигатор конкурсов и уточнение приёма работ

- первая вкладка теперь **"Конкурсы"**;
- список показывает №, название, количество работ, судей, принятых голосов, режим авторства и дату обновления;
- один клик выбирает конкурс, двойной клик открывает рабочую вкладку;
- добавлены кнопки "Открыть выбранный", "Новый конкурс", "Удалить конкурс", "Обновить список";
- приём одной работы сверён с Delphi-примером: автор назначается отдельно, тема отдельно, полный текст хранится в содержимом работы.

## STAGE12 - Contest Dashboard

Первая вкладка `Конкурсы` теперь работает как обзорная панель: выбираешь конкурс и сразу видишь карточку конкурса, рейтинг работ и контроль голосующих без перехода по вкладкам.

Сценарий:
1. Открыл программу.
2. На вкладке `Конкурсы` выбрал нужный конкурс.
3. Сразу видишь, сколько работ, судей, принятых голосов, должников и неизвестных голосующих.
4. Двойной клик открывает рабочую вкладку `Голосование`.
5. Кнопка `Итоги` ведёт сразу в финальный протокол.


## STAGE13 - импорт конкурса из текста

На первой вкладке **Конкурсы** добавлена кнопка **Импорт конкурса из текста**.

Что умеет новый импорт:

- создать новый конкурс из одного большого вставленного текста;
- определить номер и название конкурса, если они есть в тексте;
- найти ведущего и строку срока/дедлайна для журнала;
- импортировать список работ через уже существующую логику `01 - Название - Автор`;
- поддержать два режима: ведущий знает авторов или счётчик работает до раскрытия;
- автоматически добавить авторов в обязательные судьи;
- если в тексте уже есть голосования, сразу принять их в базу нового конкурса;
- перед созданием показать предпросмотр: название, режим, количество работ, судей и голосов.

Сценарий:

1. Открыл вкладку `Конкурсы`.
2. Нажал `Импорт конкурса из текста`.
3. Вставил объявление, список работ или ленту голосований.
4. Нажал `Предпросмотр`.
5. Нажал `Создать конкурс`.
6. Программа создаёт конкурс и открывает вкладку `Голосование`.

## STAGE14 - хранение в SQLite-базе проекта

Локальное JSON-хранение заменено на базу данных SQLite.

Где лежит база:

```text
Rhymers/Database/Rhymers.db
```

При запуске из Visual Studio приложение поднимается из `bin/Debug/...` вверх по папкам, находит `Rhymers.csproj` и создаёт папку `Database` прямо рядом с проектом. Если проект не найден, база создаётся рядом с исполняемым файлом.

Что хранится в базе:

- список конкурсов;
- настройки правил голосования;
- работы конкурса;
- список обязательных голосующих/судей;
- принятые голоса;
- настройки путей Excel-шаблона и итоговой папки.

Начиная со STAGE15, старые конкурсы переносятся не из JSON, а через импорт Firebird `*.fdb`.

Новая кнопка на вкладке **Голосование**:

- **Открыть базу проекта** - открывает проводник и выделяет файл `Rhymers.db`.

## STAGE15 - импорт старых Firebird *.fdb + проверка работ

По просьбе убрана JSON-миграция: старые конкурсы теперь предполагаются в Firebird `*.fdb`, а рабочая база приложения остаётся в папке проекта:

```text
Rhymers/Database/Rhymers.db
```

Что добавлено:

- кнопка **Импорт старой *.fdb** на вкладке **Конкурсы**;
- импорт из старых таблиц RhymeMachine/Firebird: `CONCURSES`, `CONCURSES_NAMES`, `CONCURS_DTL`, `POETS`, `RHYMES`, `CONCURS_VOTES`, `VOTES`;
- импорт конкурсов, работ, авторов, голосующих и старых голосов;
- повторный импорт той же `*.fdb` обновляет уже импортированные конкурсы, а не плодит дубли;
- база `Rhymers.db` остаётся рядом с проектом;
- перед приёмом одной работы и перед массовым импортом работ запускается лёгкая проверка текста.

Проверка работ ловит только нужное для приёмки:

- пустой заголовок или пустой текст;
- смешанную русско-латинскую раскладку;
- незакрытые кавычки и скобки;
- длинное тире и кавычки-ёлочки, если нужен единый конкурсный формат;
- лишние пробелы, табы, подозрительное повторение знаков;
- несколько частых опечаток вроде `джентельмен` -> `джентльмен`, `всёж` -> `всё ж`, `путкты` -> `пункты`.

Если проверка нашла предупреждения, программа показывает отчёт и даёт выбор: **вернуться исправить** или **принять всё равно**.

## STAGE16 - безопасный импорт старых Firebird `*.fdb`

- импорт старой базы теперь идёт через предпросмотр;
- можно выбрать галочками нужные конкурсы;
- перед записью создаётся backup `Database/Backups/*.db`;
- защита от дублей работает по импортному ID и, при включённой опции, по паре `номер + название`;
- голоса можно заменить импортированными или слить с уже существующими;
- при импорте применяется правило самоголоса: автор за свою работу получает `0`, если это включено в правилах конкурса;
- JSON-хранение не возвращалось: рабочая база остаётся `Rhymers/Database/Rhymers.db`.


## STAGE17 - отчёты после импорта и контрольные пакеты

Добавлен рабочий слой отчётности:

- после импорта старой `*.fdb` автоматически создаётся папка отчёта в `Database/Reports`;
- сохраняется `import_summary.txt` с параметрами импорта, backup-файлом, добавленными/обновлёнными конкурсами и предупреждениями;
- для каждого импортированного конкурса сохраняются отдельные файлы:
  - `_final_protocol.txt` - итоговый протокол;
  - `_rating.csv` - рейтинг работ;
  - `_vote_control.csv` - контроль голосующих, должники, неизвестные, пропуски;
  - `_votes.csv` - принятые голоса с отметками правил;
- на вкладке `Контроль голосования` добавлена кнопка `Сохранить отчёт контроля`;
- на вкладке `Итоги` добавлены кнопки `Сохранить пакет отчётов` и `Открыть отчёты`;
- имена отчётных файлов делаются латиницей, чтобы Windows/архивы/Excel не спотыкались о кириллицу;
- JSON не возвращался, рабочая база остаётся `Rhymers/Database/Rhymers.db`.
